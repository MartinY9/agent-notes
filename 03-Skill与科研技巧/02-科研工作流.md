# 科研工作流：让 Agent 参与整条研究流水线

Agent 在科研场景里的角色，这两年已经悄悄变了。它早就不只是帮你查查资料的搜索框，而是能实打实地参与到一整条研究流水线里来。这一篇我想结合几个最常见的科研环节——文献综述、数据分析、论文写作、同行评审——给你讲讲怎么把 Agent 真正用起来，以及配套该上哪些 Skill 和 MCP。

有一点先说明：下面提到的那些 Skill 名字，比如 literature-review、academic-research-skills，来自 2026 年公开的 Claude 和 Codex 的 Skill 市场，具体安装以对应仓库为准。但你不用太纠结名字，因为我真正想传递的是流程思路，换任何一个支持 Skill 的 Agent 都通用。

## 文献综述

先说文献综述，这大概是科研里最适合交给 Agent 的一环。目标很明确：从一个主题出发，产出一份结构化、而且带可验证引用的综述。这里可以用上一个叫 literature-review 的 Skill，它能跨 PubMed、arXiv、bioRxiv、Semantic Scholar 这些库做系统检索，生成带验证引用的 Markdown 或 PDF，支持多种引用格式，甚至能配图。

落地的流程大致是这样的。你先把数据源接上，可以用 MCP 接你已有的 Zotero 文献库，也可以让 Skill 直接去检索公开库；接着按关键词和时间范围检索，把重复的合并去掉；然后是关键的一步，让它按主题把文献聚类归纳，而不是流水账式地一篇篇罗列；每一组再去提炼核心观点、方法、结论和彼此的分歧点；最后按你指定的引用格式，比如 APA 或者国标 GB/T 7714，输出初稿，并且逐条附上来源方便你核验。

这里我要敲一下黑板：**引用务必人工核对。** 大模型是会"幻觉"的，它可能编出一篇根本不存在的文献，或者张冠李戴。带验证功能的 Skill 能把这个风险压低，但它替代不了你亲自抽查的责任。这条底线，做科研的你一定守住。

## 数据分析

数据分析是另一个 Agent 特别能打的地方，因为它天然带客观反馈。目标就是把原始数据一路带到图表和结论，让 Agent 当你的数据助手。这活儿最好用 Claude Code、Qwen Code 这种能直接跑代码的终端 Agent 来干。

流程上，你先把数据丢给它、把目标说清楚，比如"这是 data.csv，帮我看数据质量、做描述统计、找出和 Y 相关的因素"。它会自己写并运行 Python，读数、清洗、做探索性分析、画图，然后把报错和结果当作反馈来自我纠错。你在旁边看着中间结果，随时追加要求——换个模型、加个特征、改张图。最后它交给你的，是一套清洗脚本加图表，再加一份可复现的分析笔记。

Agent 之所以在这里好用，正是因为数据分析有客观反馈：报错、统计量、图对不对，这些都是明确的对错信号，让它能自己把活兜住，这恰好是它的强项。不过统计口径、因果还是相关、有没有数据泄露这些专业判断，仍然得你来把关，所以最好让它把每一步的假设都写清楚，方便你复核。

## 论文与报告写作

写作这一环，目标是把你的想法、结果和文献，变成结构清晰的稿子。这里可以用 academic-research-skills 这类成套的流水线 Skill，它通常覆盖五个环节——深度文献研究、结构化写作、批判性审稿、系统性修订、最终格式化，基本把从下笔到定稿的路走完了。

我建议的节奏是这样：先让它基于你的要点和文献生成一个大纲，你来调整结构；结构定了再一节一节地写，随时把你的实验结果和图表喂进去；初稿出来之后，让它做一次"批判性自审"，专门找逻辑漏洞、没有支撑的论断、前后矛盾的地方；最后按目标期刊或会议的模板去格式化，处理引用和图表编号。

写作这块有条红线绕不开，就是学术诚信。**AI 生成的内容需要你实质性地把关，并对最终结论负责**，同时遵守你所在机构和期刊对 AI 使用的披露要求。把 Agent 当成一个高效的助手，而不是一个能署名的作者，这个分寸请务必拿捏好。

## 同行评审的辅助

最后说说审稿。目标是更系统地审读一篇稿子，可能是你自己的，也可能是帮别人做预审。思路是用批判性审稿类的 Skill，让 Agent 按评审的几个维度——创新性、方法严谨性、实验充分性、写作清晰度、可复现性——逐项过一遍。你把全文交给它（大上下文模型一次就能塞进去），让它按一个 checklist 逐条评审、标注出具体位置的问题，最后生成一份"问题清单加修改建议"，你据此改稿或者写审稿意见。

不过这里有个特别要小心的地方：如果你是要用它来做正式审稿，一定注意期刊对"把稿件上传给第三方 AI"的保密政策。很多期刊是明令禁止的，违反了可能构成学术不端。所以这个用法，只在允许的场景、并且是对你自己的稿子时使用。

## 给科研党的一套起步组合

把上面的东西收个尾，我给你配一套起步包。主力 Agent 选 Claude Code（强）或者 Qwen Code（免费又中文）；文献库用 Zotero 加 MCP 接进来；检索靠 literature-review 这类多库检索加验证引用的 Skill；分析用能跑 Python 的终端 Agent 配数据类 Skill；写作和审稿则交给 academic-research-skills 那套成套流水线。

最后再把三条铁律给你钉一遍，因为它们真的比任何工具都重要：引用必须人工核验，遵守学术诚信和期刊的 AI 政策，以及让每一步的假设都可复核。守住这三条，Agent 才是你科研路上的助力，而不是隐患。
