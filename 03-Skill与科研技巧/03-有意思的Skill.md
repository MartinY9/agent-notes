# 有意思的 Skill 合集

科研之外，Agent 配上 Skill 还能玩出一大堆既提效又有点意思的花样。这一篇我不按部就班地讲，就当是带你逛一圈"技能集市"，挑几个 2026 年好用又好玩的方向说说，你看到感兴趣的直接去试就好。

## 文档处理三件套：PDF、Excel、PPT

第一个我想推荐的方向，是文档处理。Agent 配上文档类的 Skill，能真正地生成和操作真实文件，而不只是给你一段文本——因为它背后是在跑 Python 脚本。比如它能把一堆 PDF 里的表格抽出来、整理成结构化数据，能程序化地生成带数据透视表、图表和条件格式的多 sheet Excel 工作簿，也能从一份大纲或报告一键生成演示稿。

举个最接地气的用法：你可以直接跟它说"把这三份 PDF 报价单里的价格表抽出来，做成一张对比 Excel，最贵和最便宜的高亮出来"，它就真能给你一个成品文件。财务、运营、咨询，任何天天跟表格和报告打交道的人，都会觉得这个方向真香。

## 浏览器自动化：让 Agent 当"操作员"

第二个方向更有想象力一点。所谓 browser use 这类 Skill，能把 Agent 从一个"写代码的"变成一个能打开浏览器、点按钮、填表单、抓数据的端到端操作员，相当于一个 QA 工程师、调研分析师和自动化操作员的合体。它能帮你定期抓某些页面的数据整理成表，能自动走一遍网站流程做回归测试，也能在竞品或资料调研时批量打开、抽取、汇总。

不过用它的时候有两点要留神：一是遵守目标网站的 robots 协议和服务条款，二是小心登录态和敏感操作，别让它在你的真实账号上乱点一气。

## 办公协作的自动化

第三个方向是把跨应用的琐事串起来。这类工作流 Skill 能读你的 Gmail、起草回复、更新 Sheets、建日历事件、从一句话生成 Docs，一条指令跑完整条链路。你完全可以对它说"读今天没回的邮件，草拟回复放草稿箱，再把里面提到的会议都加进我日历"。当然，凡是涉及"发送"和"删除"这种动作，务必保留人工确认，一来防手滑，二来防提示注入。

## 给专业开发者的数据管道 Skill

如果你是数据工程师，还有一类专门的 Skill 值得关注，它能让 Agent 按照生产规范去搭建和修改数据管道——ETL/ELT、Airflow、dbt、Kafka、数据建模、数据质量校验这些都在射程之内，典型的比如 Tinybird 那类数据工程 Skill。它的价值在于，让 Agent scaffold 管道的时候能遵守你团队的约定，而不是自作主张。

## 任何编程 Agent 都值得装的通用增强

还有一批 Skill 属于"不管你用哪个编程 Agent 都值得装"的通用增强。代码审查类的能按规则审你的 diff，揪出 bug 和可以简化的地方；测试生成类的能帮你补单测、跑通、覆盖边界；提交和 PR 规范类的能生成规范的 commit message 和 PR 描述；可视化类的能把数据变成配色规范、明暗主题一致、还照顾无障碍的好看图表；安全审查类的则能扫一扫你改动里的常见安全问题。这些能力其实很多已经内置在 Claude Code 这类产品里了，比如 /code-review、/security-review，你也能在 Skill 市场找到加强版。

## 最值钱的玩法：把自己变成 Skill 作者

聊了这么多现成的，我最想安利给你的，其实是自己写 Skill。因为 Skill 最大的价值从来不是"下载别人的"，而是把你自己反复在做的事固化下来。做法很简单：先找一个你每周都要手动做好几遍的任务，可能是发周报、整理数据、格式化文档；然后写一份 SKILL.md，讲清楚什么时候用、分几步、要注意什么，需要的话配个脚本或模板；接下来让 Agent 在遇到这类任务时自动调用它。这么一来，你的经验就从此可复用、可分享给团队了。

有意思的是，这正是 Hermes 那类 Agent 想自动帮你做的事——从经验里自动长出 Skill。但说实话，你手动写往往更准、更可控，所以别把这件事全指望自动化。

## 去哪找 Skill

最后说说货源。找 Skill 主要有三条路：一是各种 Skill 市场和目录，2026 年有好几个，Skills Directory、Awesome Skills、ClawHub 都算，从科研、文档、数据到 CRM、社媒、金融，成千上万个任你挑；二是产品自带的，先看看你在用的 Agent 内置了哪些，很多需求它自己就满足了；三是 GitHub，搜 awesome agent skills 或者 claude skills，社区整理了一大堆。

但不管从哪找，有一条底线请你务必记牢：**Skill 会在你的机器上跑脚本、还可能带着 MCP 凭据，所以只装来源可信的，装之前扫一眼代码。** 供应链风险是 2026 年整个 Agent 生态里最现实的坑，别在这上面栽跟头。

说到底，Skill 的意义就是让 Agent 从"通用地聪明"变成"在你这行特别懂行"。先用现成的把效率提起来，再慢慢把自己的绝活写成 Skill——这才是把 Agent 用出复利的正确姿势。
