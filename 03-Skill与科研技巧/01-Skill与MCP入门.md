# Skill 与 MCP 入门

装好一个 Agent，其实只是把车开出了车库。真正让它从一个"什么都懂一点的通用助手"变成一个"在你这行特别懂行的专家"的，是两个机制——Skill 和 MCP。这两个词你在各家文档里都会反复撞见，这一篇我就把它们讲透，顺便说说怎么用、以及有哪些坑别踩。

## 先用一句话把它俩分清楚

很多人一开始会把 Skill 和 MCP 搞混，其实区分它们只需要一句话：**MCP 是给 Agent 接一个新工具或数据源，让它多出一项能力**，比如原本它不能查数据库，接上 MCP 之后就能了；**而 Skill 是教 Agent 把某一类专业任务做好的方法论加脚本**，比如告诉它一篇文献综述该怎么一步步做。

打个比方，MCP 像是给一个新员工发一台新设备，让他多了一项本事；Skill 则像是给这个员工一份"这活儿的标准作业流程"，让他知道怎么把事做漂亮。所以你会发现，这俩不是二选一的关系，实际用起来经常是配合在一起的。

## Skill 具体怎么用

先说说一个 Skill 长什么样。它通常就是一个文件夹，核心是一份叫 SKILL.md 的说明文件，里面写清楚两件最要紧的事——什么时候该触发这个技能，以及触发之后具体怎么做。除此之外，文件夹里还可以放一些辅助脚本、报告模板、引用格式、参考资料等等。Agent 遇到匹配的任务时，会把对应的 Skill 按需加载进来，用完就放下，既不占平时的上下文，又能在需要时格外专注。

那这些 Skill 从哪来呢？最省事的是产品自带的，Claude Code、OpenClaw 这些都内置了一批，很多时候够用了。想要更多，可以去 Skill 市场淘，2026 年冒出了好几个目录和市场，像 Skills Directory、Awesome Skills、ClawHub，从科研、文档、数据到 CRM、社媒、金融，成千上万个现成技能随你挑。但我个人觉得最有价值的玩法，其实是**自己写**——把你或者你团队反复在做的某件事，写成一份 SKILL.md，它就成了你专属的技能，从此可复用、可分享。这等于把你的最佳实践固化成了 Agent 能随时调用的东西，回报是复利式的。

用 Skill 有两条原则我想提醒你。一是**来源可信才装**，第三方 Skill 里可能夹带恶意脚本，装之前务必看一眼代码和来源。二是**小而专胜过大而全**，一个 Skill 只聚焦一件事、把触发条件写清楚，Agent 才用得准；你要是塞一堆边界模糊的技能进去，反而会让它挑花眼、选错。

## MCP 具体怎么用

MCP 的概念前一章讲过了，它是 Agent 和外部工具、数据之间的统一接口标准。你启用一个 MCP Server，支持 MCP 的 Agent 就能即插即用地调用它。典型能接的东西不少：开发方向有 GitHub、GitLab、文件系统、数据库、Sentry；科研方向有 Zotero、arXiv、Semantic Scholar 这类文献检索；效率方向则有 Slack、飞书、Notion、Google Workspace、浏览器自动化等等。

配置的思路大同小异，以 Claude Code 为例，MCP Server 一般是通过一个配置文件来声明的——你给出它的启动命令、参数，以及需要的凭据比如某个 token，配好之后 Agent 就会自动发现这个 Server 暴露出来的工具，并在需要时调用。具体的配置格式各家略有不同，看各自文档里的 MCP 章节就行，主流的 Claude Code、Codex、Qwen Code、Cline 都支持。

这里有个安全点必须单独强调：**MCP Server 常常需要凭据，这等于你把对应系统的权限交给了 Agent。** 所以只接你信任的 Server，凭据尽量用最小权限、并且能随时吊销的那种，别图省事给一把万能钥匙。

## 一个把两者用在一起的例子

光讲概念容易空，我给你举个 Skill 加 MCP 配合的完整场景。假设你想让 Agent "基于我 Zotero 里的文献，写一篇关于某主题的综述初稿"。这时候，MCP 负责的是"接上 Zotero"这一步，让 Agent 能读到你的文献库；而文献综述那个 Skill 负责的是"怎么做"——去重、按主题聚类、提炼观点、按你指定的格式生成带验证的引用，最后产出一份 Markdown 或 PDF。两者一配合，Agent 就从一个只会聊天的工具，变成了一个能交付像样综述初稿的助手。这条流程我在 [科研工作流](02-科研工作流.md) 里会讲得更细。

## 几个常见的误区

最后帮你避几个坑。有人觉得"装的 Skill 越多越强"，这是错的，装太多、触发条件又模糊，反而让 Agent 选错技能。也有人以为 MCP 和 Skill 得二选一，同样是误解，它们解决的是不同问题，通常一起用。还有人图方便，第三方的 Skill 和 MCP Server 拿来就装，这就危险了——它们都会在你的机器上真实地跑，来源一旦不可信，就是引狼入室。记住这几条，你就能把这两个机制用得又爽又稳。
