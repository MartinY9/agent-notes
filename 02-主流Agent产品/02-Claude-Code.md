# Claude Code：如果只让我留一个编程 Agent

如果你问的是"那个叫 claude 的东西到底是哪个"，答案多半就是它——**Claude Code**，Anthropic 官方出品、跑在终端里的编程 Agent。说句掏心窝的话，这套博客本身就是我用 Claude Code 一篇篇写出来的，所以这一篇我写得格外有底气。在我用过的一圈里，它是综合手感最好的那个，如果规定只能留一个，我会留它。

## 它到底是个什么东西

Claude Code 本质上是一个命令行 Agent。你在终端里跟它说人话，它就直接钻进你本地的代码库里干活：搜代码、改文件、跑命令和测试、提交 git，需要时还能联网查资料、通过 MCP 调用外部工具。背后驱动它的是 Anthropic 的 Claude 系列模型，2026 年的主力是 Opus 4.8 加上 Sonnet 和 Haiku 几档搭配着用。

它虽然叫"Code"、也确实住在终端里，但别被这个印象框住——它同时还有桌面 App、网页版（claude.ai/code），以及 VS Code 和 JetBrains 的 IDE 扩展，底层都是同一个 Agent，只是换了个入口。你完全可以在终端里起活，再切到编辑器里看 diff。要用一句话概括它的气质，我会说它是一个"贴着真实工程流程走的 AI 结对程序员"——它不满足于给建议，而是把活真正干完。

## 怎么装

整个流程其实就三步——**先备好会员账号，再装，最后首次运行时登录**。我按这个顺序说。

**第一步，得先有个能用的账号，这也是绕不开的花钱环节。** Claude Code 没有免费白嫖的路子，你得先备好下面二选一：一个**付费的 Claude 订阅**（Pro、Max、Team、Enterprise 都行，官方最推荐这条），或者一个 **Anthropic Console 的 API 预付额度**账号（按 token 计费，适合想精细控成本的）。想彻底零成本的话，那得去看国产那几家了。这一步为什么要先做？因为最后登录时登的就是它，没有账号，装好了也用不起来。

**第二步，装。** 官方现在其实更推荐"原生安装"，尤其 Windows 上比 npm 那套干净、还能自动后台更新。Windows 就在 PowerShell 里跑：

```powershell
irm https://claude.ai/install.ps1 | iex
```

或者用 winget 也行：`winget install Anthropic.ClaudeCode`。macOS / Linux 则是：

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

当然，经典的 npm 全局安装依然完全可用，前提是你有 **Node.js 18 或更新的版本**：

```bash
npm install -g @anthropic-ai/claude-code@latest
```

走 npm 这条路有个我踩过、也见无数人踩过的坑：**千万别图省事加 `sudo`**。`sudo npm install -g` 是权限报错和安全隐患的头号来源。真遇到权限问题，正确姿势是给 npm 换一个自己的全局目录，而不是提权：

```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH   # 建议把这行写进 ~/.bashrc 或 ~/.zshrc
```

**第三步，首次运行、登录，就能用了。** 装完在你的项目目录里直接敲 `claude`，第一次它会引导你在浏览器里完成登录授权，登的就是第一步那个账号；凭据登一次就存下来了，之后不用反复登。想换账号或重新登录，在会话里敲 `/login` 就行。

这里要特别澄清一个很多人以为的事——**你并不需要先手动配置什么文件才能开跑**。装完直接登录就能用，那些 `CLAUDE.md`、`.claude/settings.json` 之类是**可选的进阶配置**（用来固化项目约定、调权限、接 MCP 等），是"等你想让它更懂你项目时"才去弄的东西，不配一样正常用。这块我在下面和 [Skill 与 MCP 入门](../03-Skill与科研技巧/01-Skill与MCP入门.md) 里会讲。

至于升级，原生安装和 winget 装的会自动更新（winget 偶尔手动 `winget upgrade Anthropic.ClaudeCode` 一下），npm 装的则重新跑一遍上面那条 `@latest` 命令即可，别用 `npm update -g`，那玩意有时候会把你留在旧版本上。最后提醒一句网络：不管哪种装法，Claude 服务本身都要能访问 Anthropic，国内换镜像只能解决装包速度，解决不了服务连通，这一点得心里有数。

## 上手之后长什么样

进了交互模式，你就直接拿自然语言使唤它，不用记什么命令格式。比如让它先摸清一个陌生项目："帮我看看这个项目是干嘛的，给我个概览"；或者派它干具体的活："把 utils.py 里的日期解析函数补上单元测试并跑通"；再或者贴一段报错问它这是怎么回事。如果你想在脚本或 CI 里非交互地用，加个 `-p` 就行，比如 `claude -p "总结当前 git diff 的改动"`。

真正让它好用的是几个内置机制，值得你花点时间熟悉。**计划模式**（Plan Mode）能让它先给方案、不动手，你点头了再执行，改大东西之前特别踏实。项目根目录那份 `CLAUDE.md` 相当于一张"项目说明书"，它每次都会自动读，你可以把项目约定、常用命令、代码风格写进去，省得每次重复交代。斜杠命令则是一堆现成的快捷操作，像 `/init` 帮你生成 CLAUDE.md、`/review` 帮你审 PR。再往深了玩，还能加载 Skill 技能包、通过 MCP 接外部工具、把大任务拆给并行的子 Agent，甚至用 Hooks 在特定时机自动跑你指定的命令。这些进阶玩法我在 [Skill 与 MCP 入门](../03-Skill与科研技巧/01-Skill与MCP入门.md) 里会细讲。

## 其实更多人是在 VS Code 里用它

前面为了讲清楚原理，我一直拿终端说事，但说句实在话，日常真正高频的用法，其实是在 VS Code 里装上 Claude 的插件、边看代码边对话。我自己绝大部分时间也是这么用的——纯终端那套更多是给脚本化、CI 或者远程服务器场景准备的，坐在电脑前写代码时，编辑器里的体验要舒服得多。

装法很简单，在 VS Code 的扩展市场里搜 Claude Code（Anthropic 官方那个）装上，它会在侧边栏开一个对话面板，用的还是你同一个 Claude 账号，跟命令行是同一个 Agent、同一套能力，只是换了个更顺手的壳。

顺手在哪呢？关键就是**"看着代码聊"这件事变得自然了**。你在编辑器里选中一段代码，直接在对话框里说"把这段重构一下"或者"这里为什么会报错"，它就带着上下文去改；它做的每一处改动都会以 diff 的形式清清楚楚摆在你面前，你一眼就能看出它动了哪几行，满意就采纳、不满意就打回，完全不用像在终端里那样靠脑补去想它改了什么。它要跑命令、要修改文件，也都在你眼皮底下发生，那种"我在指挥、它在执行、我随时能叫停"的掌控感，是编辑器这个形态最大的好处。

所以我的建议是，如果你平时就在 VS Code 里写代码，那就优先用插件这条路，把它当成一个坐在你旁边、你说一句它改一片、改完还把 diff 递给你看的结对搭档；终端那套留着应急和自动化就好。JetBrains 系列（IDEA、PyCharm 等）也有官方插件，思路完全一样，喜欢哪个编辑器就用哪个。

## 它适合你吗

我的建议是这样：如果你有一定工程底子、习惯或者至少不排斥命令行，又经常需要 Agent 深入真实代码库做多文件、多步骤的改动，那 Claude Code 几乎是闭眼入的选择。搞科研和数据的朋友也很值得上，配合 Skill 做文献、跑分析、写论文，效率提升相当明显，这条线我单独写了一篇 [科研工作流](../03-Skill与科研技巧/02-科研工作流.md)。

要说它不太适合谁，主要是两种人。一种是预算极其敏感、就想彻底白嫖的，那 Claude Code 的付费门槛绕不过去，可以去 [国内产品](07-国内AI编程产品.md) 那篇找免费额度友好的。另一种是习惯了"边打字边自动补全、点两下就完事"那种传统 AI 编程体验的人——虽然上面说了 VS Code 插件已经把命令行的门槛降得很低，但它骨子里还是个"你下指令、它对话式地干活"的 Agent，而不是那种深度嵌进编辑器的行内补全；如果你更想要后者，国产的 Trae、通义灵码或者开源的 Cline 会更对路。

## 关于钱和安全

收费这块前面说过了，靠付费订阅或 API Key，订阅按月含一定额度，API 按 token 走。重度使用要留个心眼盯着用量，尤其 Opus 这一档单价不便宜。

安全上它默认对危险操作是要确认的，我个人的习惯是——**在不可回滚的目录里，绝不轻易开全自动**。另外让它去读外部网页或来路不明的文件时，多留个心眼防提示注入；API Key 也务必收好，别手滑提交进 git。这些老生常谈，但真出事都是栽在这上面。
