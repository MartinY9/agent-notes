# OpenClaw：一个周末项目，怎么就成了 GitHub 顶流

在一堆一本正经的编程 Agent 里，OpenClaw 是个异类，也是 2026 年最值得讲的一个故事。它不写代码为主业，而是奔着"装在你自己电脑上的贾维斯"去的——一个开源、可自托管、能替你打理各种杂事的个人 AI Agent。而它爆红的速度，放在整个开源史上都算得上现象级。

## 一个奥地利工程师的周末，和 34 万颗星

故事的主角叫 Peter Steinberger，一位奥地利工程师。2026 年 1 月，他把自己一个周末捣鼓出来的开源 AI Agent 扔上了网，本来没当回事，结果 48 小时内这个项目冲上了十万星，几个月后累计突破 34 万星，一度是整个 GitHub 上最热的仓库。热到什么程度？它后来直接转成了非营利组织运营，有了全职团队。一个周末项目走到这一步，你就知道它戳中了多少人的痒点。

它戳中的那个痒点，其实就是大多数 AI 助手做不到的一件事——**主动替你动手**。别的助手是你问一句它答一句，OpenClaw 则是把大模型直接接到你本机的文件、应用、浏览器和系统工具上，你给个简单的聊天指令，它就能读写文件、操作软件，真正把事情办了。它自带一百多个内置 Skill，还有一个庞大的 Skill 市场（比如 ClawHub）可以扩展，能连上 WhatsApp、Discord 这类聊天软件，7×24 地在后台帮你自动化。而且它是模型无关的，你可以自带云端模型的 API Key，也可以让它完全跑本地模型，隐私攥在自己手里。

## 怎么用（说个大概）

因为它是自托管项目，安装方式会随版本一直变，具体请以官方仓库的 README 为准，我这里只把大致流程给你捋一遍，让你心里有数。

拿到项目之后（从官网 openclaw.ai 或 GitHub 上的 `openclaw/openclaw`），你首先要给它配个大脑，也就是填入某家云端模型的 API Key，或者指向本地的 Ollama 之类。接下来最关键的一步是**授权**——你要告诉它能访问哪些目录、哪些应用、哪些浏览器和消息渠道。我特意把这一步单拎出来强调，因为这一步本质上就是在给它划能力和风险的边界，划到哪它就能干到哪，也就能闯祸到哪。配好权限，再从内置或市场里挑几个你需要的 Skill 装上，然后就可以用自然语言、甚至直接在聊天软件里给它派活，它会在后台持续替你跑。

正因为它能量大、又跑在你自己机器上，官方和社区都反复强调同一句话：**先在隔离、低权限的环境里试，确认它行为可控了，再一点点放权。**

## 它适合谁

这么说吧，如果你想要的是一个管生活、管自动化的全能助理，而不只是一个写代码的工具，那 OpenClaw 就是当下生态最火、玩法最多的选择。如果你还格外看重隐私和本地化、又不怕自己动手部署，那就更对味了。反过来，你要是只想安安静静写代码，那 [Claude Code](02-Claude-Code.md) 或 [Codex](03-OpenAI-Codex.md) 会专注得多；要是你没准备好承担自托管那份安全责任，也建议先别急着上。

## 认真说说安全，这一节请务必读完

OpenClaw 的强大，直接来自"它能碰你的真实世界"，而这份强大也让它在 2026 年成了当年**第一个重大 AI Agent 安全事件**的主角。我不想吓唬你，但这几个风险是实打实的，值得你正眼看。

最直接的是**过度授权**：一旦你把聊天软件、文件、浏览器都接上，那么它被滥用或被诱导时，波及的就是你真实的账号和数据。紧接着是**提示注入**——它读到的网页、邮件、消息里可能藏着恶意指令，诱使它去执行危险动作，这类攻击对能"读了就动手"的 Agent 尤其致命。还有**供应链风险**，你从市场里装来的第三方 Skill 可能夹带私货，装之前一定看清来源和代码。最后别忘了它能执行本机命令，这意味着它有造成不可逆破坏的能力。

那怎么办？我的务实建议就几条：最小授权，能不给的权限就别给；隔离环境里跑；只装可信来源的 Skill；对"发送消息、删除、转账"这类动作坚持保留人工确认；再定期翻翻它的行为日志。把这几条做到位，你才能安心享受它的强大。

顺带一提，OpenClaw 主打对 Claude 的 Agent Skills 兼容，和 Claude 那套 Skill 概念高度互通，你完全可以拿 Claude 当它的大脑——毕竟它模型无关，谁强用谁。
