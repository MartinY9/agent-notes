# 国内 AI 编程 Agent：中文友好、额度大方、网络无忧

聊完了国际上那些产品，这一篇我特别想留给国内的朋友。因为坦白讲，前面那些官方和开源 Agent 再强，对国内用户都有两个绕不开的小坎——网络得折腾，还常常要掏外币。而国产这一挂，恰恰把这两个坎给填平了：中文交流自然、免费额度大方、访问没有任何障碍。到了 2026 年，国内的 AI 编程工具已经进入"Agent 协作时代"，产品形态铺满了网页版、IDE 插件、AI 原生 IDE 和命令行工具四条线，热闹得很。下面挑几家主流的说说。

## Qwen Code：我最看好的"国产版 Claude Code"

如果只让我在国产里推一个给爱用命令行的人，那一定是阿里的 Qwen Code。它是一个开源的终端优先编程 Agent，从 Gemini CLI 的代码基改造而来、为 Qwen3-Coder 模型做了优化，而且是 Apache 2.0 开源的，态度很敞亮。能力上该有的都有——理解大型代码库、改文件、跑命令、审代码、自动化 Git、通过 MCP 接外部工具，模型上也不挑食，OpenAI、Anthropic、Gemini、Qwen 乃至本地的 Ollama、vLLM 都能接。它还额外给了 IDE 插件、桌面 App、守护进程模式、SDK，甚至连 Telegram、钉钉、微信、飞书的机器人都备齐了。

安装是熟悉的 npm 一条龙（具体包名以官方 README 为准）：

```bash
npm install -g @qwen-code/qwen-code@latest
qwen
```

一句话，想要免费、开源、中文友好又是终端形态的，Qwen Code 是国内对标 Claude Code 和 Codex 的首选，没有之一。

## 通义灵码：装机量最大的那位 IDE 助手

同样出自阿里云，通义灵码走的是 IDE 内嵌路线，VS Code 和 JetBrains 都有插件，企业生态成熟、中文场景打磨得很细。它适合那种在编辑器里边写边补、顺手做点仓库级问答和改动的日常场景。装它也简单，在 VS Code 或 JetBrains 的插件市场搜"通义灵码 / Tongyi Lingma"，登录阿里云账号即可。

## Trae：字节的 AI 原生 IDE，用户量领先

Trae 是字节跳动 2025 年 1 月推出的一个 AI 原生 IDE——注意，它是一个独立的编辑器，不是插件，由 Skywork-Coder 等模型驱动，有免费版也有企业版。它的势头相当猛，公开数据说已经积累了约六百万注册、一百六十万月活，一年生成的代码量以千亿行计。你从官网下载对应平台的安装包，像装 VS Code 一样打开就能用。要我说，Trae 是国产里对新手最友好的一个，开箱即用、全程可视化、中文体验也顺，完全不用碰命令行。

## iFlow（心流）：完全免费的终端尝鲜款

iFlow CLI 内置了 GLM-4.6、MiniMax M2 这些开源模型，背后也能用上 Qwen3-Coder，最大的卖点是**完全免费**，特别适合零成本试水终端 Agent。安装参考官方给的方式，配好就能在终端里用。当然免费也有代价，公开评测里提到它在复杂项目上的代码质量还有提升空间，你可以拿它练手、跑轻量任务，别一上来就压重活。

## 腾讯、百度、智谱：各自云生态里的选手

剩下几家我放一起说。腾讯云的 **CodeBuddy** 采用混合架构，在响应速度和能力之间做平衡，也在往 AI 原生 IDE 的形态走。百度的 **文心快码（Comate）** 用了所谓"三明治"架构，把百度自家的 AI 积累深深揉进编码场景，以 IDE 插件为主。智谱的 **CodeGeeX** 则是老牌国产选手，模型自研、插件形态、多语言补全和对话都支持。这三家的安装大同小异，都是在插件市场搜对应名字装上、登录各自账号，如果你本来就在腾讯云、百度或智谱的生态里，顺手选对应的那个就好。

## 国产该怎么挑

简单收个尾。想要开源、终端、能对标 Claude Code 的，闭眼选 Qwen Code；想要开箱即用的 AI 原生 IDE，选 Trae；想要企业成熟生态、走 IDE 插件路线的，通义灵码、CodeBuddy、文心快码里挑一个；想完全免费尝鲜的，iFlow。它们共享的那三个优势——中文自然、额度大方、网络无障碍——对国内的团队和个人来说，是官方和开源产品短期内很难替代的体验。想看跨阵营的完整对比，还是那句话，翻 [横向对比与选型](08-横向对比与选型.md)。
