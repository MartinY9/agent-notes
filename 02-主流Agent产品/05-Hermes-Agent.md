# Hermes Agent：一个会自己成长的私有 Agent

如果说 OpenClaw 是"即插即用的全能助理"，那 Nous Research 的 Hermes 就是它气质相近、内核却很不一样的那个兄弟。同样是开源、自托管、能力全面，但 Hermes 押注了一个更有想象力的方向——**让 Agent 自己成长**。你用它越久，它就越懂你、越能干，甚至会从经验里自己长出新技能。这是我觉得它最迷人也最值得单独讲的地方。

先消个歧，免得你被名字绕进去：Nous 的 Hermes 既是一个开源模型系列（Hermes 3、Hermes 4 那些），也是这个 Agent 产品本身。这篇讲的是后者，也就是 Hermes Agent；它默认可以配 Hermes 系列模型，但并不绑死，换别的模型也行。

## 它凭什么"会成长"

Hermes Agent 由以开源模型和去中心化训练闻名的 Nous Research 出品，2026 年 2 月开源，定位是跑在你自己服务器上的自主 Agent，主打持久记忆加上自我成长。

真正让它区别于其他 Agent 的，是官方所说的那个"学习闭环"。按照描述，它是少见的能把这几件事串起来的 Agent：它会从使用经验里**自动创建新的 Skill**，并在往后的使用中不断打磨改进这些技能；它会**主动提醒自己**把重要的知识持久化保存下来，而不是用完就忘；它能**回头检索自己过去的对话**；它还会跨会话不断积累，慢慢建立起一套对"你到底是谁"的深层画像。除此之外，它的常规集成也相当齐全，网页搜索、页面抽取、完整的浏览器自动化、视觉分析、图像生成、语音合成、多模型推理，一应俱全。

用一句话概括这份独特：别的 Agent 需要你不停地喂 Skill，而 Hermes 会自己把 Skill 长出来。

## 怎么装

它天生就是面向自托管、要常驻跑着的，所以最主流的装法是 **Docker**，而且它比 OpenClaw 更"服务器化"——大多数人干脆就把它扔在一台便宜的 VPS 上养着。具体命令以官方文档（hermes-agent.nousresearch.com/docs）为准，我把关键步骤给你捋一遍。

先看配置门槛，其实很低：不带浏览器工具的话，1 个 vCPU、1GB 左右内存就能跑；要开浏览器自动化（Camofox 或 Chrome CDP），那就给到 2 vCPU、4GB 内存。官方镜像是 `nousresearch/hermes-agent:latest`。一个最小可用的起法大概长这样：

```bash
docker run -d --name hermes --restart unless-stopped \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent gateway run
```

这里 `-v ~/.hermes:/opt/data` 这个挂载很关键——它把配置、凭据和**会话记忆**都持久化到宿主机的 `~/.hermes` 目录，这样容器重启、镜像升级之后，它辛苦攒下来的记忆和自建 Skill 不会丢。第一次跑会有个配置向导问你要大模型的 API Key（OpenAI、Anthropic、OpenRouter、本地 Ollama 都能配，可以配多个），填完写进 `~/.hermes/.env` 就行。

装好之后，用它的心态和别的 Agent 很不一样：**让它常驻跑着别关**。因为它的价值是随运行时间累积的，记忆越攒越多、自建的 Skill 越来越顺手。所以别把它当"用完即走的工具"，而更像是"你养起来的一只私有 Agent"，你越舍得跟它长期磨合，它回报你的就越多。

## 它和 OpenClaw 到底差在哪

这俩都是自托管、个人向、能力全面的 Agent，实话说很容易混。我的理解是，差别主要在气质和侧重上。OpenClaw 主打的是主动式个人助理，强项在于连接各种生活应用、加上一个超大的 Skill 市场，社区现象级、拿来即用；而 Hermes 主打的是自我学习闭环，强项在于越用越强、记忆持久，背后还站着 Nous 那套模型和去中心化训练体系。所以选择其实很简单：你想要一个即插即用的全能助理，去 OpenClaw；你想要一个会跟着你一起成长的私有 Agent，来 Hermes。

## 它适合谁，以及要当心什么

坦白讲，Hermes 不是给图省事的人准备的。它最适合那种手里有服务器、愿意长期运维一个私有 Agent 的进阶玩家，或者对"Agent 自我进化、持久记忆"这个前沿方向本身就着迷的研究者和爱好者，当然也适合特别看重开源和本地掌控的人。如果你只是想快速写代码、或者压根不想碰运维，那它大概率不是你的菜。

安全上，自托管的老规矩一条不少——权限、浏览器自动化、命令执行，每一样都是攻击面，责任在你自己。因为它多半跑在一台独立服务器上，天然就和你的主力机隔开了，这点比装在自己电脑上的 Agent 要省心；但**别因此就掉以轻心把它裸奔在公网上**。这里有个血淋淋的教训值得记住：2026 年 6 月，就有攻击者利用 Hermes 那个**未设鉴权、直接暴露在公网的管理面板**下手——互联网扫描器扫到这些暴露的面板，把 Agent 诱导去植入了 SSH 后门。官方后来把鉴权做成了强制项（凡是非本机绑定都必须过鉴权）。教训很简单：**管理面板/端口别对公网开，能只绑本机就只绑本机，一定要暴露就套上鉴权和防火墙。**

除了网络暴露，它还有一个别家没有的特殊注意点：正因为它会自动创建和持久化 Skill，你得**定期回头审查它自己长出来的那些东西**，别让它把某个错误的经验固化成了 Skill，那样错误会被反复复用。另外它的长期记忆（就在你挂载的 `~/.hermes` 里）会沉淀不少敏感信息，这个目录的存储安全和备份也得上心。
