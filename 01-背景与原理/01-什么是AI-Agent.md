# 什么是 AI Agent（智能体）

在动手装任何工具之前，我想先花一篇的篇幅把"AI Agent 到底是什么"这件事讲清楚。因为这个词这两年被用得太滥了，什么东西都往上贴，反而把人搞糊涂了。如果非要我用一句话定义它，我会说：**AI Agent 是一个能自己规划步骤、调用工具、观察结果、再决定下一步，从而把一件事真正做完的 AI 系统——而不只是回答你一句话。** 这句话里的每个动词都是关键，下面我慢慢拆给你看。

## 从"会聊天"到"能干活"，到底变了什么

早期的大模型，比如最初的 ChatGPT，本质上是一个很聪明的"文本接龙"机器：你给一段话，它续一段话。它知识渊博、表达流畅，但有两个致命的硬伤。一是它只会"说"不会"做"，你让它帮你改代码，它只能把改法讲给你听，真正动手的还是你自己；二是它一问一答、转身就忘，不会为了一个更大的目标持续努力。

AI Agent 干的事，就是在这个"会说话的大脑"外面，套上一层由循环、工具和记忆组成的外壳，把它从一个顾问变成一个能干活的员工。这个转变有多大呢？我举个例子你就懂了。假设你说"帮我把这个项目里所有过时的 API 调用改掉并跑通测试"。一个普通的聊天机器人会给你一段"你可以这样改"的建议，剩下的苦力活全归你。而一个真正的 Agent 会自己动手：先在代码里搜出所有调用点，一个个改掉，然后运行测试，看到报错就去修，修完再跑，如此反复，直到测试全绿，最后回过头来告诉你"改完了，测试都过了"。

这就是"Agent"这个词最近突然火起来的根本原因——它把 AI 从一个动嘴的顾问，变成了一个动手的执行者。

## 拆开看，一个 Agent 由哪几块拼成

理解一个 Agent 的内部构造，最简单的办法就是拿"一个人干活"来类比。最核心的当然是大模型本身，它相当于**大脑**，负责理解你的目标、做推理和决策。但光有大脑不够，它还需要一双**手和眼睛**，也就是各种工具——读写文件、执行命令、搜索网页、调用 API，这些让它能真正接触和改变外部世界。有了大脑和手，还得有干活的**节奏**，也就是那个"想一步、做一步、看一眼结果、再想下一步"的循环，这是 Agent 区别于一次性问答的灵魂。除此之外它还需要一个**记事本**，也就是记忆，用来记住上下文、历史和你的偏好；以及一个明确的**工位和权限边界**，规定它能碰哪些文件、能不能联网、能不能执行命令。

这几块具体怎么运作、各自有哪些讲究，我在下一篇 [Agent 核心概念与架构](02-Agent核心概念与架构.md) 里会逐一展开，这里你先有个整体印象就好。

## 市面上的 Agent，大致是这么几种形态

"Agent"这个筐太大了，真正落到产品上，主要就是几种形态，认清形态能帮你迅速判断一个产品适不适合你。

最贴近开发者的是**终端里的编程 Agent**，它跑在命令行里，直接读写你本地代码库、跑测试、提交 git，Claude Code、OpenAI Codex、Qwen Code、opencode、Aider 都属于这一类。它们的优点是贴近真实开发流程、能脚本化、能进 CI，缺点是你得习惯命令行。另一种是**编辑器里内嵌的 Agent**，要么是 VS Code 插件比如 Cline，要么是一整个 AI 原生 IDE 比如 Trae、通义灵码、CodeBuddy，它们胜在可视化、上手快，代价是相对绑定编辑器。

再往外走，就跳出编程了。**自托管的个人助理型 Agent**，比如 OpenClaw 和 Hermes，能连接你的文件、浏览器、聊天软件，帮你打理生活和自动化任务，能力全面、隐私可控，但配置和安全责任都得你自己扛。最后还有一类**自主任务型 Agent**，典型是 OpenHands，你给个高层目标它就自己拆解执行，最接近大家想象里的"数字员工"，自动化程度高，但也更需要你盯着、成本和不可控性都更高。

## 一段简短的来龙去脉

顺便捋一下这几年是怎么走到今天的，你会对整个领域更有感觉。2022 年底 ChatGPT 引爆了大模型，但那时还是纯对话。2023 年，ReAct、AutoGPT、LangChain 这些把"推理加工具加循环"的范式跑通了，"Agent"概念第一次走红，只可惜当时效果不稳定，经常自己绕圈子。到了 2024 年，模型的工具调用能力大幅增强，官方的 CLI Agent 开始出现，Agent 这才真正变得可用。2025 年，MCP 这个连接工具和数据的协议成了事实标准，编辑器内嵌的 Agent 和国产 AI 编程工具大规模普及。而 2026 年，Skill 这种可复用技能和 Agent 市场兴起，个人 Agent 和会自学的 Agent 相继出圈，Agent 终于从一个"编程工具"扩展成了"通用执行者"。

## 三条我希望你一开始就有的清醒认知

Agent 很强，但它绝不是万能的，有三件事我希望你从一开始就记在心里。第一，**它会犯错，而且会自信地犯错**，所以删文件、改数据库、对外发消息这种关键操作，一定要留一个确认或审查的环节。第二，**你给它的权限，就是它的攻击面**，那些 API Key、命令执行权、文件读写权，别人或者一个恶意网页也可能借它的手来用，OpenClaw 在 2026 年就因为安全问题上过热搜。第三，**它最擅长那些有反馈的任务**，写代码能跑测试、有报错，反馈闭环好，它就表现好；反过来那些主观的、无法验证对错的任务，它就很容易跑偏。

想清楚这三点，你再去用任何一个 Agent，心态都会稳很多。
